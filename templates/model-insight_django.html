System Architecture Overview

Our system employs a multi-model approach for disaster tweet analysis. The architecture consists of several key components that work together to analyze and classify tweets:

Text Preprocessing

Tweets are cleaned and normalized using NLP techniques to remove noise and extract meaningful features.

Feature Extraction

TF-IDF vectorization converts text into numerical features that can be processed by machine learning algorithms.

Primary Classification

Logistic Regression model determines if a tweet is about a real disaster with high accuracy.

Ensemble Learning

Random Forest classifier provides additional validation and improves prediction accuracy.

Sentiment Analysis

VADER Sentiment Analyzer measures the emotional tone of tweets to provide context about the severity.

Entity Recognition

Named Entity Recognition with spaCy identifies locations and other important entities mentioned in tweets.

Prediction Function

Here's a simplified version of our tweet classification function that predicts whether a tweet is about a real disaster:

def predict_disaster(text): # Vectorize the text features = vectorizer.transform([text]) # Scale the features scaled_features = scaler.transform(features.toarray()) # Make prediction with our trained model prediction = lr_model.predict(scaled_features)[0] probability = lr_model.predict_proba(scaled_features)[0][1] # Return result as dictionary result = { "prediction": "Disaster" if prediction == 1 else "Not a Disaster", "probability": float(probability) } return result

Our entity extraction function identifies locations mentioned in tweets:

def extract_locations(tweet): # spaCy entity recognition doc = nlp(tweet) locations = [ent.text for ent in doc.ents if ent.label_ == "GPE" or ent.label_ == "LOC"] # Regular expression patterns for location extraction patterns = [ r"\b\w+(?: city| town| village| state| province| country)\b", r"\b\w+(?: mountain| river| lake| ocean| sea)\b", r"\b[A-Z][a-z]+, [A-Z]{2}\b" # US city, state format ] for pattern in patterns: regex_locations = re.findall(pattern, tweet) locations.extend(regex_locations) return list(set(locations)) # Remove duplicates
